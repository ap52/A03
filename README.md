# A03
Webstorm Setup
Step 1 - download webstorm https://www.jetbrains.com/student/

Step 2 - install git https://git-scm.com/downloads

Step 3 - connect your github account to webstorm through system preferences -> Version Control then enter the path to git.exe

Step 4 - add password to webstorm go to system prefrences -> appearance and behavior -> system settings -> passwords

Step 5 - go to your github account and create a repository to connect to webstorm

Step 6 - Go to VCS and import the repository, enter the repository name and local path name

Step 7 - in webstorm go to file -> new -> HTML, add this to the local file system

Step 8 - Commit your changes and push go to git -> push

Step 9 - File should be in github, in github go to settings and scroll down to github pages change the source to master branch and then your site url should be present

Step 10 - Check the URL in your brower to check if it works 





Glossary:

Branch - a set of code that is under development without affecting the production / main branch code

Clone - copies an existing repository 

Commit - saves your work with the ability to log changes you made

Fetch - adding changes from the remote repository to a local branch without commiting them 

GIT - version control software

Github - source coding hosting service

Merge - takes changes from one branch and applies them to another

Merge Conflict - unable to fix difference in code between commits

Push - send commited changes to a remote repository  

Pull - fetching changes and merging them 

Remote - a repository or branch that is hosted on a server

Repository - contains a projects files
